# 포맷 및 설정 상수

IBS_InitSheet()의 Cfg 나 Cols 속성에 설정 가능한 값을 설명합니다.


## IBS_InitSheet Cfg 또는 SetConfig 함수 설정 상수

### (Cfg)SearchMode (조회 모드)

| 상수 | 값 | 설명 |
|------|------|------|
| `smLazyLoad` | 2 |LazyLoad 조회 (기본, 권장) - 스크롤 시 렌더링 |
| `smServerPaging2`| 4 | 서버 페이징 조회 - 대량 데이터 |

- **일반/중소 데이터**: `smLazyLoad` 사용 (기본, 권장)
- **대량 데이터 (10만건+)**: `smServerPaging2` + `DoSearchPaging()` 사용

---

### (Cfg)MergeSheet (머지 모드)

| 상수 | 값 | 설명 |
|------|-----|------|
| `msNone` | 0 | 머지 없음 |
| `msAll` | 1 | 전체 머지 |
| `msPrevColumnMerge` | 2 | 헤더머지X + 데이터 영역 앞 컬럼 기준 머지 |
| `msFixedMerge` | 3 | 고정 머지(MultiRecord사용시) |
| `msHeaderOnly` | 5 | 헤더만 머지 |
| `msPrevColumnMerge` + `msHeaderOnly` | 7 | 헤더 머지 + 데이터 영역 앞 컬럼 기준 머지 |

---

## (Cfg)CountPosition (건수 표시 행 위치)

| 값 | 설명 |
|-----|------|
| 0 | 미표시 |
| 1 | 좌측 상단 |
| 2 | 우측 상단 |
| 3 | 좌측 하단 |
| 4 | 우측 하단 |
---

## (Cfg)AutoFitColWidth (특정 시점에 각 컬럼 너비 재조정)

| 값 | 설명 |
|-----|------|
| 'search' | 조회 종료 후 |
| 'resize' | sheet width resize 시 |
| 'init' | ibsheet7객체 생성 시점 |
| 'colhidden' | 컬럼 보임/감춤 시점 |

---

## IBS_InitSheet Cols 또는 InitColumns 함수 설정 상수

### (Cols)SumType (합계 유형)

| 값 | 설명 |
|-----|------|
| `"Sum"` | 합계 (기본) |
| `"Avg"` | 평균 |
| `"Count"` | 건수 |


---

### (Cols)Format (데이터 포맷)

#### 숫자 포맷

| 값 | 설명 | 예시 |
|-----|------|------|
| `"Integer"` | 정수 (#,###) | 1,234 |
| `"NullInteger"` | 정수(0은 빈값) | |
| `"Float"` / `"NullFloat"` | 실수 | 1,234.56 |
| `"#,###"` | 사용자 정의 숫자 | 1,234 |
| `"#,##0.00"` | 소수점 2자리 | 1,234.00 |
| `"#,###만원"` | 접미어 | 1,234만원 |
| `"####년"` | 접미어 | 2024년 |
| `"#,##0\\%"` | 퍼센트 | 75% |

#### 날짜 포맷

| 값 | 설명 | 예시 |
|-----|------|------|
| `"Ymd"` | 날짜 (YYYY/MM/DD) | 2024/01/15 |
| `"Md"` | 날짜 (MM/DD) | 01/15 |
| `"Hm"` | 시간 (HH:MM) | 14:30 |
| `"yyyy-MM-dd"` | 날짜 커스텀 | 2024-01-15 |

#### 특수 포맷

| 값 | 설명 | 예시 |
|-----|------|------|
| `"PhoneNo"` | 전화번호 | 010-1234-5678 |
| `"PostNo"` | 우편번호 | 123-456 |
| `"IdNo"` | 주민번호 | 901231-1****** |
| `"CardNo"` | 카드번호 | 1234-5678-9012-3456 |

### 사용자 정의 포맷 (배열)
```javascript
// 주민번호: 개인/법인 자동 판별
{Format:["######-#******","###-**-**###"]}

// 단일 사용자 정의
{Format:"###-**-**###"}
```
